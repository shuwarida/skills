## What it does

`pirate-mode` is a development philosophy switch: move fast, skip the bureaucracy, reach the goal by the shortest honest route. The agent stops asking permission for reversible things and just ships — while talking like a pirate. Confident, bold, no hedging, no "maybe" or "you might consider". Yo-ho-ho.

Underneath the tone there is a real workflow — eight steps from **Pirate Code** (project charter) through **Battle Plan** (technical design) to **The Plunder** (execution). None of the steps are mandatory: the mode explicitly skips whatever is overkill for the size of the task. For a small job, a manifest and the plunder are enough.

## When to reach for it

Say **"pirate mode"** or **"pirate dev mode"**, or just ask to work fast without ceremony — the agent activates it on its own when a prototype or demo is wanted without unnecessary formalities.

Reach for it when speed of getting to a working result matters more than process: prototypes, demos, spikes, hackathon-style work. The focus is MVP, not the perfect solution — if something can be cut, it gets cut.

## Pirate's Honor: fast never means reckless

A real pirate doesn't sink their own ship. Speed applies only to what's reversible. Before anything destructive or irreversible — deleting data, force-pushing, rewriting history, touching prod, spending money — the mode drops anchor and asks first. Speed is for building, never for breaking.

## Exiting the mode

Ask to return to the standard approach — the mode switches off explicitly, it doesn't expire on its own.
